Optional streamingProfile

streamingProfile: boolean

Inherited from IJsonLdParserOptions.streamingProfile

  • Defined in packages/actor-rdf-parse-jsonld/node_modules/jsonld-streaming-parser/lib/JsonLdParser.d.ts:130

If this parser can assume that parsed documents follow the streaming JSON-LD profile. If true, and a non-streaming document is detected, an error may be thrown. If false, non-streaming documents will be handled by preemptively buffering entries, which will lose many of the streaming benefits of this parser.

Concretely, if true, @context definitions must come as first object entries, followed by @type (if they define a type-scoped context).

Defaults to false for spec-compliance.

streamingProfileAllowOutOfOrderPlainType: boolean

Inherited from IJsonLdParserOptions.streamingProfileAllowOutOfOrderPlainType

  • Defined in packages/actor-rdf-parse-jsonld/node_modules/jsonld-streaming-parser/lib/JsonLdParser.d.ts:205

When the streaming profile flag is enabled, @type entries MUST come before other properties since they may defined a type-scoped context. However, when this flag is enabled, @type entries that do NOT define a type-scoped context may appear anywhere just like a regular property.

Optional strictValues

strictValues: boolean

Inherited from IJsonLdParserOptions.strictValues

  • Defined in packages/actor-rdf-parse-jsonld/node_modules/jsonld-streaming-parser/lib/JsonLdParser.d.ts:164

By default, JSON-LD requires that all properties (or @id's) that are not URIs, are unknown keywords, and do not occur in the context should be silently dropped. When setting this value to true, an error will be thrown when such properties occur.

This also applies to invalid values such as language tags.

Defaults to false.

Optional validateValueIndexes

validateValueIndexes: boolean

Inherited from IJsonLdParserOptions.validateValueIndexes

  • Defined in packages/actor-rdf-parse-jsonld/node_modules/jsonld-streaming-parser/lib/JsonLdParser.d.ts:181

If @index inside array nodes should be validated. I.e., nodes inside the same array with the same @id, should have equal @index values. This is not applicable to this parser as we don't do explicit flattening, but it is required to be spec-compliant. Defaults to false.

Spec-compliance: to be fully spec-compliant, this must be explicitly set to true.
